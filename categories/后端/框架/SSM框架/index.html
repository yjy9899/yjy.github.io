SSM框架

2022-11-19 面试 简历, 总结, 面试 0

SSM框架介绍

1.SSM框架是Spring、Spring MVC 、和mybatis框架的整合

spring:实现业务对象管理;Spring MVC:页面的跳转请求和对具体的视图解析，而处理不了的请求则交给tomcat服务器来处理。;Mybatis:数据对象的持久化引擎。

2.标准的SSM框架有四层，分别是dao层（mapper），service层，controller层和View层。

  • 持久层/dao层（mapper）层:主要是做数据持久层的工作，负责与数据库进行联络的一些任务都封装在此。定义接口的实现类。
  • 业务层/Service层:主要负责业务模块的逻辑应用设计。业务逻辑层的实现具体要调用到自己已经定义好的Dao的接口上，建立好Dao之后再建立service层，service层又要在controller层之下，因为既要调用Dao层的接口又要提供接口给controller层。每个模型都有一个service接口，每个接口分别封装各自的业务处理的方法。
  • 表现层/Controller层（Handler层）:负责具体的业务模块流程的控制。
  • View层:主要负责前台jsp页面的表示。

3.DAO层，Service层这两个层次都可以单独开发，互相的耦合度很低，完全可以独立进行，这样的一种模式在开发大项目的过程中尤其有优势，Controller，View层因为耦合度比较高，因而要结合在一起开发，但是也可以看作一个整体独立于前两个层进行开发。

Spring介绍

1.Spring是一个轻量级Java开发框架。它是一个分层的JavaSE/JavaEE 一站式轻量级开源框架，简化Java开发的复杂性。解决的是业务逻辑层和其他各层的松耦合问题，因此它将面向接口的编程思想贯穿整个系统应用。

2.Spring是一个基本框架，同时提供容器来装载具体的bean对象。没有Spring前，创建对象是用New；有了Spring之后，只需要告诉Spring容器有哪些对象，它会帮我们创建和维护整个对象的生命周期，生态比较完善。

2.Spring的核心框架

  • IOC（控制反转）：把原来创建对象的过程交给了Spring容器进行控制和管理。

    • IOC重点：在系统运行中，动态的向某个对象提供它所需要的其他对象。这一点是通过DI（依赖注入）来实现。DI使得对象之间的耦合度降低，提高了代码的可维护性和可测试性。Spring提供了三种常见的依赖注入方式：构造函数注入、Setter方法注入和字段注入

    它可以装载bean，也是一种降低对象之间耦合关系的设计思想。（比如租房子。以前租房子需要一个房子一个房子找，费时费力，然后现在加入一个房屋中介，把你需要的房型告诉中介，就可以直接选到需要的房子，中介就相当于spring容器。）

  • AOP（面向切面编程）：面向对象开发的一种补充，它允许开发人员在不改变原来模型的基础上动态的修改模型以满足新的需求，即不修改源代码进行功能增强。通常情况下，与业务无关的逻辑代码可以通过AOP实现，例如：日志、事务管理、权限控制等。在需要进行动态扩展的方法上采用AOP的五大通知：前置、环绕、后置、异常、最终，添加与业务无关的逻辑，从而完成某个功能。

    • AOP可以将一些横切关注点（如日志、事务、安全等）从应用程序的核心逻辑中抽离出来，使得代码更加模块化和可维护。Spring AOP基于代理模式实现，可以在运行时动态地将横切逻辑织入到目标对象的方法中。

4.Spring的优点

  • 轻量级和非侵入性：Spring框架是一个轻量级的框架，它不需要大量的依赖库和容器。它不会强制你的应用程序继承特定的类或实现特定的接口，因此可以与现有的应用程序无缝集成。
  • 控制反转（IoC）和依赖注入（DI）：Spring采用了控制反转和依赖注入的设计模式，使得对象之间的依赖关系更加松散，并且更容易管理。这种设计模式有助于提高代码的可测试性、可维护性和可扩展性。
  • 面向切面编程（AOP）：Spring提供了AOP支持，使得开发者可以将横切关注点（如事务管理、日志、安全等）从核心业务逻辑中分离出来，从而实现更好的模块化和可维护性。
  • 方便和其他框架进行整合:Spring框架提供了广泛的集成支持，可以与各种其他框架和技术（如Hibernate、JPA、MyBatis、Struts、JSF等）无缝集成.

Spring MVC介绍

1.SpringMVC是Spring框架的一个模块，用于构建基于MVC（Model-View-Controller）设计模式的Web应用程序。

2.SpringMVC的工作流程

  1. 用户发送请求至前端控制器DispatcherServlet。
  2. DispatcherServlet收到请求调用HandlerMapping处理器映射器。
  3. 处理器映射器找到具体的Handler处理器(可以根据xml配置、注解进行查找)，生成处理器对象及处理器拦截器(如果有则生成)一并返回给DispatcherServlet。
  4. DispatcherServlet调用HandlerAdapter处理器适配器。
  5. HandlerAdapter经过适配调用具体的处理器(Controller，也叫后端控制器)。
  6. Controller执行完成返回ModelAndView。
  7. HandlerAdapter将controller执行结果ModelAndView返回给DispatcherServlet。
  8. DispatcherServlet将ModelAndView传给ViewReslover视图解析器。
  9. ViewReslover解析后返回具体View视图。
  10. DispatcherServlet根据View进行渲染视图（即将模型数据填充至视图中）。
  11. DispatcherServlet响应用户。

SpringMVC的工作流程-简单版

  1. 用户发送请求至前端控制器DispatcherServlet。
  2. DispatcherServlet 根据请求信息调用 HandlerMapping，解析请求对应的 Handler（处理器）。
  3. Handler 执行业务逻辑，将处理结果封装为 ModelAndView 对象。
  4. DispatcherServlet 调用 ViewResolver 解析 ModelAndView 对象对应的 View（视图）。
  5. 最终将结果响应给客户端。

3.SpringMVC常用的注解

  • @RequestMapping：指定请求路径，用于建立请求 URL 和处理请求方法之间的对应关系。可用于类和方法上，用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。
  • @RequestBody：用于建立请求 URL 和处理请求方法之间的对应关系。可用于类和方法上，用于类上，表示类中的所有响应请求的方法都是以该地址作为父路径。
  • @ResponseBody：实现将Controller方法返回对象转化为json对象响应给客户。
  • @SessionAttributes:加在类上，指定哪些属性名，或者哪些属性值会被加入到session中

mybatis介绍

1.是一个用于简化数据库操作的持久层框架，在内部封装了通过JDBC访问数据库的操作,在开发时，只需关注SQL语句本身，不需要花费精力去处理驱动加载，连接创建等繁杂的过程。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集的过程，减少了代码的冗余，减少程序员的操作。

2.MyBatis的核心思想是将SQL语句与 Java 对象之间的映射关系配置在XML文件或注解中，从而将SQL语句与Java代码分离，这样就可以在不修改代码的情况下直接在配置文件中修改SQL，使得数据持久化操作更加灵活和易于维护。

3.mybatis的操作都是围绕一个sqlSessionFactory实例展开的。mybatis通过配置文件关联到各实体类的Mapper文件，Mapper文件中配置了每个类对数据库所需进行的sql语句映射。

4.mybatis的缓存

mybatis的缓存分为一级缓存和二级缓存，一级缓存放在session里面，默认就有。二级缓存存放在命名空间里，使用二级缓存属性类实现Serializable序列化接口，可在它的映射文件中配置。
