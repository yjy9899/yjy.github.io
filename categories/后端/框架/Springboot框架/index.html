SpringBoot框架

2022-11-19 面试 简历, 总结, 面试 0

SpringBoot框架介绍

SpringBoot是简化了Spring应用的的初始搭建及开发过程的框架，是Spring组件一站式的解决方案。它只需“run”就能创建一个独立的、生产级别的Spring应用。

SpringBoot框架特性或优点

  • 快速创建独立的Spring应用
  • 直接嵌入应用服务器，如Tomcat,jetty,undertow,不需要去部署war包
  • 大量的自动配置，简化开发

SpringBoot配置文件的格式？它们有什么区别

提供两种文件格式的全局配置文件，并且文件名是固定的，均以application为文件名，格式分别是application.properties和application.yml的配置文件

  • application.yml：这种配置文件是基于YAML语言的文件

SpringBoot设置理念

设置理念：约定优于配置

约定优于配置，也称作按约定编程，是一种软件设计范式，旨在减少软件开发人员需做决定的数量，获得简单的好处。比如：

  • 默认有resources文件夹,存放资源配置文件。src-main-resources,src-main-java
  • 默认有target文件夹，将生成class文件盒编程生成的jar存放在target文件夹下
  • spring boot默认的配置文件必须是，也只能是application.命名的yml文件或者properties文件，且唯一

SpringBoot最重要的两个功能？

自动配置 Auto Configuration

Spring Boot 根据应用程序配置文件和已有的 Spring Bean 来自动配置应用程序。自动将一些配置类的bean注册进ioc容器，我们只需要在需要使用的地方用@Autowired或者@Resourse注解来注入。这使得开发者无需手动配置大量的 Spring 组件，而是可以根据需要进行定制化配置。
